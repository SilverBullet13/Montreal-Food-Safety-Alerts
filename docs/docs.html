/contrevenants

/contrevenants get

get

Returns either the full list of violations that occurred between two given dates or a list of total violations by establishment during specified period.

get /contrevenants

Returns either the full list of violations that occurred between two given dates or a list of total violations by establishment during specified period.

  • Request

Query Parameters

  • 200: required(string)
  • 400: required(string)

    Invalid or missing date parameters.

  • du: required(string)

    Start date (format YYYY-MM-DD)

    Example:

    2023-01-01
  • au: required(string)

    End date (format YYYY-MM-DD)

    Example:

    2023-12-31
  • sum: required(string)

    If true, returns the sum of violations by establishment. Otherwise, returns full details.

/contrevenants/total-violations

/contrevenants/total-violations get

get

Returns a list of all establishments with one or more known violations. Each item includes the number of violations. Results are sorted in descending order of total violations. Supports both JSON (default) and XML (if 'format=' query parameter is used).

get /contrevenants/total-violations

Returns a list of all establishments with one or more known violations. Each item includes the number of violations. Results are sorted in descending order of total violations. Supports both JSON (default) and XML (if 'format=' query parameter is used).

  • Response

HTTP status code 200

Body

Media type: application/json

Type: array of object

Items: ViolationSummary

  • etablissement: required(string)
  • total: required(integer)

Media type: application/xml

Type: string

Example:

<?xml version="1.0" encoding="UTF-8"?>
<violations>
  <etablissement>
    <nom>BOULANGERIE LA TUNISIENNE</nom>
    <total>3</total>
  </etablissement>
</violations>

Media type: text/csv

Type: any

Example:

etablissement,total
BOULANGERIE LA TUNISIENNE,3
PALAIS MAHARAJA,2

/demand-inspection

/demand-inspection post

post

Submits an inspection request for an establishment. The request body must contain client information and the observed issue.

post /demand-inspection

Submits an inspection request for an establishment. The request body must contain client information and the observed issue.

  • Request
  • Response

Body

Media type: application/json

Type: InspectionRequest

HTTP status code 201

HTTP status code 400
